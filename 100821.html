jQuery (part 1) – Friday, Oct. 8, 2021

Alright, alright, it's week 5. By now we know the drill. The lovely sources this blog references are the jQuery API documentation and ole reliable, a W3 schools tutorial. Soooo, what is jQuery? Simply put, it is a JavaScript library. Without JavaScript there is no jQuery. jQuery simplifies JavaScript. JavaScript and jQuery sitting in a tree, W-E-B-D-E-V-E-L-O-P-I-N-G. First and foremost, to have a copy of jQuery in your code, it must be included within <script> tags:

<body>
    <script src="jquery.js"></script>
    <script>  
        // code goes here      
    </script>
</body>

Note that the downloaded jQuery file should be in the same directory as the pages using it. The beauty of jQuery is that it takes common, lengthy tasks of JavaScripts and wraps them into methods. Essentially, in jQuery you query HTML elements and perform some action on that element. To do so, the syntax is $(selector).action()

The $ accesses jQuery, the (selector) queries/finds the desired HTML elements, and lastly .action() is the jQuery action to be performed. For example: $(".student").show() shows all HTML elements with the class "student". For the most part, jQuery uses CSS to select elements, so a strong knowledge of CSS and JavaScript are needed before diving deep into jQuery.

In order to prevent jQuery from running before the document is loaded, it's common to explicitly indicate when to run the actions by nesting jQuery within a ready function:

$(document).ready(function(){
    // jQuery methods go here      
});

So now we've ensured that our document is fully loaded before we perform actions on it. Phew! Now, how do we trigger these actions we want to perform? jQuery can fire an action based off of a list of events. Events can include a mouse click, keypress, scrolling, a form submit, all of those nice DOM events. Below is an example of how a function is passed to an event. In this case, when a user double clicks the button element, the element is hidden:

$(".button").dblclick(function(){
    $(this).hide();
});

Notice how similar that code looks compared to the ready document event. We've seen examples of the .show() and .hide() effects, but jQuery offers a broad range, such as:

  • toggle()- toggle between showing and hiding an element
  • fade()- fade an element into and out of visibility, also fadeIn() and fadeOut()
  • slideDown()- slide an element down. Can also slideUp()
  • animate()- create a custom animation for an element
  • stop()- stop an animation before it's finished

And so much more. Too much jQuery goodness to cover in a single blog post– come back next week for part 2 to learn about manipulating HTML, traversing HTML, and maybe even AJAX. TTYL, HTML!
