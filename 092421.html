Bootstrap – Friday, Sep. 24, 2021

What is Bootstrap? Well, it's not just a strap at the back of a pair of boots. Bootstrap is a UI framework that takes a mobile-first web development approach (hey, remember that from the last blog post?). So let's begin to understand how it helps by taking a look at the Bootstrap documentation.

Bootstrap is a library of UI components, content formats, forms, and other such layout tools. Many of its components require JavaScript plugins, so it's necessary that the <script> tag be specified within the <body>. Similarly to what we discussed in Week 2, Bootstrap also recommends following a template which meets development standards, includling the <meta> tags. Everything's coming full circle! We'll start by loading the Bootstrap CSS into our <head>, before all other styling sheets:

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.1.1/dist/css/bootstrap.min.css" rel="stylesheet"
    integrity="sha384-F3w7mX95PdgyTmZZMECAngseQB83DfGTowi0iMjiWaeVhAn4FJkqJByhZMI3AhiU" crossorigin="anonymous">

By default, Bootstrap employs a grid system. As such, it offers three main types of containers, the default of which (.container) is responsive, with its max-width changing at each breakpoint. So rather than concernng ourselves with CSS styling for a container, Bootstrap does the work for us (given we have all the correct imports in place to use it):

<div class="container">
    <!-- Content here -->
</div>

You'll notice that breakpoints are a big deal in Bootstrap. A breakpoint is essentially a designer-specified width that determines how responsive the layout is across device screens/viewport widths. These breakpoints can also be called grid tiers, ranging from extra small to extra extra large. This breakpoint sizing helps uphold consistency by way of scaling for common device/viewport dimensions. Columns are also available within Bootstrap and make use of the grid system, as its hierarchy system goes container → row → column → content. Both Bootstrap's grid and column layouts rely on a flexbox architecture.

One area of Bootstrap I have experience in is its form control options; by default, Bootstrap builds a form vertically, but there are also horizontal and inline form layouts. Bootstrap's global styling for forms means that most form labels and controls should be wrapped in <div class="form-group"> and that .form-control should be added to all input and selection items. In this example we are also using class="mb-3" to have a margin bottom (mb) on each element. Here's what the code would look like for a vertical form that asks for a username, password, and a checkbox if the user wants their credentials to be saved, as well as a login button: 

<form>
    <div class="mb-3">
        <label for="userID" class="form-label">Username</label>
        <input type="username" class="form-control" id="userID" aria-describedby="userNameHelp">
        <div id="userNamelHelp" class="form-text">Your Duke's email eID</div>
    </div>
    <div class="mb-3">
        <label for="userPW" class="form-label">Password</label>
        <input type="password" class="form-control" id="userPW">
    </div>
    <div class="mb-3 form-check">
        <input type="checkbox" class="form-check-input" id="rememberUser">
        <label class="form-check-label" for="remmeberUser">Remember me next time</label>
    </div>
    <button type="login" class="btn btn-primary">Login</button>
</form>
