I'll be honest-- I miss CSS. Secretly I think I'm right-brained, because what's not to love about visual stuff!? So this blog is going to be my opportunity to explore more of the wacky and wonderful whims that CSS can fulfill, particularly with image filters. Let's take this picture of our robot friend here and see what we can do with it:

For starters, CSS has this beautiful thing called the filter property that we can use to specify visual effects like grayscale, contrast, blur, opacity, etcetera, which is typically used on images.

Here are some of the options that filter styling offers us:

  • blur(px)- blurs the image by a pixel amount. The larger the pixel, the more blurred
  • contrast(%)- depending on the %, increases or decreases contrast. Default contrast is 100%; lower reduces contrast, higher increases the contrast
  • grayscale(%)- % is optional. Used to determine the balance of black & white
  • hue-rotate(deg)- probably to coolest one. Rotates the hue around the color circle as specified by the degree.
  • invert(%)- inverts the image from 0-100%. 100% is completely inverted
  • sepia(%)- applies a sepia filter as strong as the specified %

With these, we can try some new styles on our robot friend!

Above, we are applying a grayscale, lowering brightness, inverting, adding a sepia filter, and then blur filter, in that order. This just looks like defining a filter for the image style, such as style="filter:brightness(65%)". Remember how I said hue-rotate is the coolest one? Let's see what it looks like to rotate in increments of 90 degrees around the color wheel:

Swanky! That's achieved by setting style="filter:hue-rotate(0deg)" and increasing the degvalue by 90 each time. But that's not all we can do! We can apply multple filters at once, too:

This is achieved by simply tacking one filter after another:

<img src="https://i.pinimg.com/564x/95/c1/f2/95c1f28ae26fd250572becf8080e7ece.jpg" class="blogimg"
    style="filter:hue-rotate(120deg) invert(1) contrast(150%) drop-shadow(8px 8px 10px cyan)">

But I wanted to learn more about the logic of hue-rotate. I did something similar in java my freshman year but have since then forgotten the logic behind this. While reading more on the hue-rotate function, I found that its official syntax is:

hue-rotate() = hue-rotate( [ <angle> | <zero> ]? )

Where angle determines how much to rotate the angle. This could be in the terms of degrees, radians, gradians, or turns. We can see from the graphic above that red starts at angle 0. In the terms of our robot, who starts as yellow-orange (~45 degrees, according the wheel), incrementing by 90 brings us to ~135, which is green. The above examples of the robot using the hue-rotate filter reflects these findings! Oddly enough, the Internet Explorer browser does not support this type of CSS.
