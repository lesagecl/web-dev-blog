Templates & SEO – Friday, Sep. 10, 2021

After a quick Google search of "How to write better HTML", the article I found basically advised front-end web developers to avoid falling into the routine of <div> and <span>, and rather to make use of the templates offered by HTML. Two benefits of doing so included "Search Engine Optimisation, of course" and that "Template structures are more identifiable and therefore more readable."

Search Engine Optimization? Of course! Wait... what's Search Engine Optimization (SEO)? According to HTML documentation, it refers to the process by which developers make a website appear higher up in results from a search engine. Working on the SEO improves a website's traffic, as more users tend to click on the first few search results they see. As Google is the most popular search engine, a website optimized for Google will perform well, allowing for more search traffic.

For example, Google's search algorithm favors mobile-friendly websotes over those that are not. The more portable your website is, the more traffic you may get!

The ultimate goal of search engines like Google is to return results that are relevant, accessible, and high-quality. SEO can be achieved in a variety of ways. A couple of these are by implementing Google Analytic tools–which track site metrics–and by having other websites backlink your own, thus growing your credibility. But I wanted to focus on the HTML-specific qualities of SEO.

For example, the <meta> tag is used to define metadata about key concepts of an HTML document. Though this metadata does not appear on the screen, it is used by browsers, search engines, and other webservices to indicate the content of your webpage. Consider the following code:

<head>
 <meta charset="UTF-8">
 <meta name="description" content="Week 2 blogpost">
 <meta name="keywords" content="HTML, SEO, web traffic">
 <meta name="author" content="Chelsea Le Sage">
</head>

The charset attribute indicates which character encoding is used for the document. Meanwhile, the name attribute can indicate a few values, like an author, description, or key words for the document. Including these tags contribute to SEO!

So now, using that code for metadata we can follow a typical HTML template, because recall that templates improve readability:

<!doctype html>
 <html>
 <head>
  <title>Web Dev Blog</title>
  <meta charset="UTF-8">
  <meta name="description" content="Week 2 blogpost">
  <meta name="keywords" content="HTML, SEO, web traffic">
  <meta name="author" content="Chelsea Le Sage">
 </head>
 <body>
  Here's some blog content!
 </body>
 </html>

This template structure is important because <div> and <span> are not semantic HTML elements, meaning that they do not tell the computer anything about their content; they are too general purpose. But keep in mind that semantic tags, however, do not offer any style specifications. So with semantic tags providing us document structure, meta tags specifically supplying metadata to search engines, and with a mobile-portable website, traffic should start flowing! Just make sure not to let the fame get to your head.
