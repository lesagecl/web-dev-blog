The Inspect Tool – Friday, Sep. 3, 2021

During development, us programmers have to put on our investigator hats; or rather, today we'll be wearing our inspect-igator hat. Sure, Chrome provides official documentation that gives clear step-by-step instructions for using their DevTools and Inspect feature, but why would you read that when you can listen to me, someone with a brand-new domain name and a week-and-a-half of web development under their belt?

We'll be focusing specifically on the Chrome Inspect tool because that is the browser I prefer to use. It's a feature I learned to use long before ever studying computer science– in middle school, we used it to change the text on the official county website for various, devious reasons. But nowadays I use it to find color values, certain font names, and understand web layouts, no longer to convince my mom that it actually is a snow day.

My go-to way to access this feature is to right click while on a web page and select "Inspect" from the drop-down, which opens the Chrome DevTools panel. Then click the Inspect icon:

Now, clicking on an element will show us where in the Document Object Model (DOM) tree that object occurs. Essentially, this DOM tree outlines the logical structure of our HTML, with its 'cascading' format. HTML tags are treated as objects, and anything nested within a tag is its child. This makes things simple for pinpointing specific webpage elements. For example, I visited twodee.org and felt quite inspired. I inspected the text beneath the postTitle (CS 347: Lab 3 – More HTML) and found a tag I've never seen before, the <em> tag. This is a tag for emphasized text. Well, that doesn't seem too crazy, because we can use <i> to italicize the text and get the same thing right? Wrong!


The <em> tag is especially cool because with the use of a screen reader, any words within the tab will be verbally stressed. This way, we can make our message more accessible to any user that is visually imapaired and relies on assistence. Here, I use <em> underneath the page title to describe what the blog post is about as well as the date: <em style="font-family: 'IBM Plex Sans', sans-serif; color: #7e7e7e; font-size: small;">The Inspect Tool – Friday, Sep. 3, 2021</em>

Another cool accessibility feature is how DevTools will indicate whether or not the color contrast is large enough. Essentially, this helps users with color blindess or poor eyesight, as a higher contrast between text and its background makes it easier to read. In the screenshot above, I pulled the red/salmon color from the aforementioned postTitle. The checkmark next to the 3.71 indicates that DevTools approves of color balance.

One more feature of the Chrome DevTools I want to touch on is the ability to make stylistic changes without having to update the actual code; if I want to experiment with different colors for my text or backgrounds, I can change the values within the DevTools Styles tab and see it change without needing to save and compile code. I didn't know that until now! Though I can wrap my brain better around the UI-adjacent features of DevTools, the documentation after the DOM section dives into network activity, cookies storage, and debugging – all extremely helpful devices for when your blog traffic is more than one person a week. Who knows, maybe I'll write a post about that in the future.

Until then, DOM-inate the rest of your day!
