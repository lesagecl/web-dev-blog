Media Queries – Friday, Sep. 17, 2021

Last week we talked about Search Engine Optimization, and one of the rules of optimization was having a site that is mobile-friendly. To achieve this we must use a responsive CSS technique called Media Query. During my research of this topic, two sources I consulted were Mozilla's documentation and W3 Schools.

Media queries make it so that a webpage is adaptable for multiple screen dimensions, resolutions, and so on. As part of its syntax, the media type can optionally be defined to describe the type of device the code chunk is being made suitable for. This includes all, print, screen, and speech. Let's get into some code, shall we?

Media features always evaluate to either true or false. Below showcases how media features can have a range, min or max, which defines a minimum or maximum condition. For example:

@media screen (max-width: 600px) {
    body {
        background-color: red;
    }
}

This makes it so that when the user's browser window is less than 600 pixels, the background color changes to red. Notice that above uses the screen and max-width specifications. Multiple types and features can be specified following @media for particular cases in which the web page must be able to adapt to. 

@media only screen and (min-width: 475px) and (orientation: landscape) {
    <!-- some code-->
}

Above, any style specifications would be limited to a device with a screen, whose brower window is at least 475 pixels and in the landscape orientation. One thing I learned from Chapter 7 of All Over the Web which was not discussed in the aforementioned documentation is that within your CSS file, a media query must occur after a rule set, if not to be overwritten by the rule set. Thus it makes sense to declare most media queries at the bottom of the css file. One important philosophy introduced by this chapter of the text is that of the mobile-first development approach to web pages; I agree with the logic that a web page that works well on small screens probably works well on larger screens, and not the other way around.

Connecting this newfound knowledge to my Week 1 blog about the Chrome Developer Tools, it's important to note that it can be used to preview what the web page looks like at different screen layouts; the Toggle Device Toolbar widget allows you to preview the page on various devices, from iPhone to iPad, and even those that must not be named (*shudder* Android devices).
